# 桃源镇：重置前修复 + 重置 + 验证报告

## 执行结果 (2026-06-09)

### 修复的 Bug 清单

#### 1. 技术爆炸 (严重)
- **问题**: `innovationPhase` 有 20% 自动发现概率; `DiscoveryEvents` 中 int 值(0-100)被直接作为 skillLevel(3-9)使用, 导致几乎所有 agent 都能研究最高级技术
- **修复**: 移除 `innovationPhase` 的 20% auto-discovery; `skillLevel = Math.floor(int / 10)` 映射; 移除 `effectiveProb` 倍增器; 研究员门槛从 int≥4 改为 int≥30; 季度最多发现数 2→1

#### 2. 智力归零 Bug (严重)
- **问题**: `Math.min(20, agent.stats.intelligence + 1)` 将智力 clamp 在 20, 导致高智力角色(如周晓月 int 80)在执行研究失败时有 30% 概率被降到 20
- **修复**: `Math.min(100, int + 1)`

#### 3. 生育率过高 + 不合理 (中等)
- **问题**: 基础生育率 12%/季, 49岁还能生, 无生育间隔
- **修复**: 4%/季, 最大生育年龄 42, 添加上次生育后 2 年间隔检查

#### 4. 儿童健康归零 (严重)
- **问题**: 儿童无收入无食物库存, 每季粮食不足扣 5 health 直到 0; 全村最多 3 个儿童全部 health=0
- **修复**: ① 粮食消耗系统 health 下限 0→1; ② 新增父母代喂机制(未成年人从父母库存取粮); ③ 暴风雨受伤下限 0→1; ④ LifecycleSystem 增加全局 health≥1 安全钳

#### 5. 孩子只从父姓 (轻微)
- **问题**: 所有新生儿从父姓
- **修复**: 50% 从母姓, 50% 从父姓

#### 6. 年终报告新生儿统计错误 (轻微)
- **问题**: `a.born === year` 会把初始角色(born=0)在 year 0 全统计为新生儿
- **修复**: `a.born === year && a.born > 0`
