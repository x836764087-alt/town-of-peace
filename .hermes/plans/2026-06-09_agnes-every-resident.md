# Agnes AI + 桃源小镇：每人一个AI（v2 全量接入版）

## 核心变化（vs v1）

| 维度 | v1（叙事层方案） | v2（全体居民方案） |
|------|-----------------|-------------------|
| 接入范围 | 每季1-5次LLM调用 | 每季覆盖全部居民（轮转） |
| 成本 | 有限制 | 2.0-flash 免费，无成本瓶颈 |
| 瓶颈 | cost | throughput / rate limit |
| 架构 | Batch Post-Processing | 事件驱动 + 轮转采样 |

## 核心设计：Active Agent Queue

每季的LLM调用流程：

```
        所有居民列表（50-100人）
               ↓
   ┌──────────────────────────┐
   │ Event Detector           │
   │ 识别本季"有大事"的居民    │
   │ 结婚/死亡/致富/冲突/饥饿  │
   └─────────┬────────────────┘
             ↓ 优先级A（必须处理）
   ┌──────────────────────────┐
   │ Priority Sampler         │
   │ 从剩余居民中抽样 ~20%    │
   │ 优先选：上一轮没抽到的   │
   └─────────┬────────────────┘
             ↓ 优先级B（轮转处理）
   ┌──────────────────────────┐
   │ LLM Queue Manager        │
   │ 排入队列，按批次提交API  │
   │ 每批次最多 10 个并行请求 │
   └─────────┬────────────────┘
             ↓
       生成结果写入 agents
```

**效果估算**（50人小镇）：
- 优先级A（大事触发）：~5-8人/季
- 优先级B（轮转抽样）：~8-10人/季
- 每季总LLM调用：~13-18次
- 每人平均间隔：~3-5季轮到一次
- 大事触发者下次自动提前

## 居民AI层级定义

### P-0：纯模拟（默认，新生儿初始）
- 用当前 procedural 逻辑
- 仅记录事件（无LLM）

### P-1：人格层（每人生而有之）
- 出生时LLM生成：姓名含义、性格倾向、初始记忆片段
- 人格随经历渐变（LLM定期微调）
- 存储为 `agent.persona`
```typescript
interface AgentPersona {
  traits: string[]         // 如 ["勤劳", "固执", "念旧"]
  values: string[]         // 如 ["家庭优先", "厌恶风险"]
  memories: AgentMemory[]  // 最近 5 条重要记忆
  narrative_arc: string    // 人物弧光描述
}
```

### P-2：决策层（触发时启用）
- 关键选择：职业、配偶、搬家、是否借粮
- LLM根据人格+记忆+当前处境给出3个选项及权重
- 模拟系统按加权随机选择
```typescript
// 例：赵长河（里正）是否要开仓放粮？
// LLM 输入：人格(保守, 顾全大局) + 记忆(去年旱灾) + 现状(库存80%, 饥民20人)
// LLM 输出：[
//   { action: "开仓放粮", weight: 60, reason: "顾全大局" },
//   { action: "只放一半", weight: 30, reason: "保守但仁慈" },
//   { action: "不开", weight: 10, reason: "留到更紧急时" }
// ]
```

### P-3：记忆层（每季更新）
- 本季发生的事 → LLM 压缩成 1-2 条记忆
- 每人保留最近 5 条记忆
- 旧记忆自动衰减（加权摘要）

### P-4：对话层（抽样触发）
- 选中两人 → LLM 生成情景对话
- 对话基于：关系值 + 最近记忆 + 当前活动
- 写入 chronicle，不影响模拟状态

### P-5：叙事层（每年一次）
- 每人年度故事线 → LLM 生成一段个人叙事
- 用于丰富的年末汇总（取代模板）

## 新生儿接入流程

```
新生儿出生事件
    ↓
生成初始人格（LLM）：
  - 姓名：家族姓氏 + 寓意名
  - 基础性格倾向（继承父母部分特征）
  - 初始记忆："生于桃源镇第N年，父亲是XX，母亲是XX"
    ↓
设定为 P-0（无活动能力，纯模拟）
    ↓
成年后（16岁）：
  - 自动升级到 P-1（人格完整）
  - 可进入 P-2 决策层
```

## 技术架构

```
src/llm/
  ├── llm-client.ts           # Agnes AI API 封装
  ├── queue-manager.ts        # 优先级队列 + 速率控制
  ├── agent-llm.ts            # 居民LLM交互入口
  ├── persona-generator.ts    # 人格生成/更新
  ├── decision-engine.ts      # 决策请求
  ├── memory-system.ts        # 记忆压缩/存储
  ├── dialogue-system.ts      # 对话生成
  └── event-detector.ts       # 判断谁需要LLM
```

## 速率控制策略

免费模型通常有 RPM 限制（推测 ~60 RPM）。

**策略**：批处理窗口
- 每秒最多 1 个 batch（每个 batch 10 个并行请求）
- 每季总调用 15-20 次 → ~2 轮 batch → ~2 秒 LLM 处理时间
- 加在模拟 tick 末尾，不影响主循环

## 第一阶段动手范围

先建基础设施，做通一个 agent 的全链路：

1. `llm-client.ts` — 封装 API 调用（验证 key 可用性）
2. `persona-generator.ts` — 给一个居民生成人格
3. `event-detector.ts` — 识别大事触发
4. `memory-system.ts` — 单条记忆写入/读取
5. 连起来跑一次完整 tick：居民 A → LLM 生成人格 → LLM 生成决策 → 写入模拟

通过后批量扩展到所有居民。
