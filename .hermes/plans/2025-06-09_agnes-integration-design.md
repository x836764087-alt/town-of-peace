# Agnes AI + 桃源小镇：LLM 接入方案深度分析

## 核心原则

LLM 不是替代模拟逻辑，而是补上"模拟做不到的事"——**情感、意外、意义**。

| 谁来干 | 适合什么 |
|--------|---------|
| **模拟代码（Phase 0-8）** | 确定性机制：经济、生育、健康、交易、饥饿、冲突概率 |
| **LLM（Agnes AI）** | 生成性内容：叙事、对话、异常事件、个性化描述 |

LLM 说"可能发生了什么"，模拟代码说"这件事的后果是什么"。

---

## 接入架构三层

从低成本到高成本，每层可以独立上线：

### 层 1：叙事层（Narrative Layer）— 最低成本，立竿见影

**当前问题**：chronicle 输出模板化——"张三和李四结婚了"、"今年丰收了"。读起来像日志，不像历史。

**LLM 做的事**：模拟跑完一季后，把这一季的关键事件（婚姻、死亡、贸易、冲突、科技发现）打包发给 Agnes AI，让它生成一段带情感的叙述。

```
模拟输出：                              LLM 输入：
  agent.A 与 agent.B 结婚               "桃源镇第5年春天发生以下事件：
  agent.C 饿死了                           - 张三和王婆的女儿翠花结婚
  agent.D 发明了造纸术                     - 李老四在冬天饿死了
                                           - 周建国造出了第一张纸"
                                         ↓
                                       LLM 输出：
                                       "春风里，张三迎娶了王翠花。
                                        没人注意到，李老四的屋子
                                        已经三个月没冒烟了...而
                                        周建国手里那张薄薄的纸，
                                        将永远改变这个小镇。"
```

**成本估算**：每季 1 次 API 调用，~2K tokens → 几乎免费

### 层 2：对话层（Dialogue Layer）— 中成本，让居民有"声音"

**当前问题**：`dialogue-topics.ts` 里是预设话题列表，agent 之间的"社交"只是概率选择话题，没有真正的交流。

**LLM 做的事**：每季挑选 3-5 对关系值最低/最高的 agent，基于他们的状态（年龄、职业、最近发生的事）生成一段对话。

- 张三（刚失业）遇到李四（刚发财）→ 对话内容体现张力
- 王婆子（接生婆）遇到孕妇 → 家长里短
- 赵长河（里正）召集议事 → 讨论当前最紧迫的问题

**关键设计**：对话内容不影响模拟状态（只记录在 memory 中），但可在下一轮被引用。

**成本估算**：每季 3-5 次 API 调用，每次 ~4K tokens → 低

### 层 3：事件层（Event + 蝴蝶效应层）— 中等成本，让世界有"意外"

**当前问题**：`town-events.ts` 的事件是预设模板 + RNG 触发，"盗窃"只是扣钱，"火灾"只是减健康。事件之间没有因果联系。

**LLM 做的事**：每季生成 1-2 个"微事件"（small narrative beats），有 10% 的链式概率发展成"蝴蝶效应"。

```
Layer 3 Architecture:

  [LLM 生成初始事件]
       ↓
  (示例) "陈大江在河边捡到一个 
         奇怪的金属片"
       ↓
  [模拟系统介入]
       ↓
  选项 A：不管它 → 无事发生
  选项 B：拿去铁匠铺 → 
         张大山研究后改良了农具
       ↓
  [下一季，LLM 检查链式概率]
       ↓
  10%：改良农具大受欢迎 → 
       需求量暴增 → 铁矿石紧缺 →
       价格波动 → 经济变化...
  3%：金属片是古代遗迹的一部分 →
       引发考古热潮 → 吸引外来者 →
       人口结构变化...
```

**蝴蝶效应的关键机制**：不是 LLM 决定结果，而是 LLM 生成"可能性种子"，模拟系统用规则决定是否发芽。

## 接入方式：技术方案对比

### 方案 A：Batch Post-Processing（推荐 MVP）

```
      每季模拟结束
           ↓
    收集 state 变更摘要
           ↓
    批量调用 Agnes AI（3-5 个 prompt）
       ├── narrative（1 call）
       ├── dialogues（3 call）
       └── events（1 call）
           ↓
    将输出写入 chronicle / memory
           ↓
    进入下一季
```

**优点**：
- 模拟核心不受 LLM 延迟影响
- 确定性模拟逻辑完全保留
- 成本可预测
- 可以随时开关 LLM 层

### 方案 B：Agent on Demand（高级版）

```
  关键 Agent（赵长河、周建国...）
       ↓
  当面临重要决策时（非每季）
       ↓
  构造 agent 上下文（性格、记忆、处境）
       ↓
  LLM 输出决策建议（非强制指令）
       ↓
  模拟系统按概率采纳或拒绝
```

**适用场景**：里正决定是否开仓放粮、是否宣布紧急法令、是否接纳外来者。
**限制**：只用于 2-3 个关键角色，普通居民不用。

### 方案 C：Hybrid（推荐长期）

```
  ┌─────────────────────────────┐
  │   LLM 叙事层 (narrative)     │ ← 每季批量
  │   LLM 对话层 (dialogue)      │ ← 每季批量
  │   LLM 种子事件 (event seeds) │ ← 每季批量
  └──────┬──────────────────────┘
         │ 只读模拟状态
         ↓
  ┌─────────────────────────────┐
  │   模拟核心 (Phase 0-8)       │ ← 确定性，所有决策在此
  │   - 经济、生育、交易          │
  │   - 冲突、法律、科技          │
  │   - 位置、资源、饥饿          │
  └──────┬──────────────────────┘
         │ 输出事件结果
         ↓
  ┌─────────────────────────────┐
  │   LLM 叙事层 (wrap-up)       │ ← 把结果写成故事
  └─────────────────────────────┘
```

## 自然世界系统：让生态"长起来"

你的想法 "让自然界的生物长起来"——这个需要明确边界。

### 轻量版（推荐）

**定义**：在现有地图上增加资源再生/消耗逻辑 + LLM 叙事包裹。

```
每个季节：
  1. 森林：产出木头量 = base × 天气修正
     - 如果砍伐 > 再生 → 森林缩小（tile 类型变化）
  2. 水域：产出鱼量 = base × 季节修正
     - 过度捕捞 → 产量下降
  3. 野生动物：简单数量模型
     - 兔子：每季繁殖 +2（条件：food > threshold）
     - 狼：每季 -1（条件：兔子数量 < 5）
  4. LLM 叙事："今年冬天特别冷，河里的鱼少了很多，
     兔子也不怎么见踪影。张大山打了三天猎，
     只带回一只瘦弱的野兔..."
```

**地图变化可见**：森林 tile→荒地 tile、荒地 tile→森林 tile（如果人不干预）

### 重量版（备选，远期）

- 完整食物链模型（植物→食草→食肉）
- 物种迁移（气候变化影响栖息地）
- 季节迁徙路径（候鸟、鱼群）
- 入侵物种（通过贸易偶然引入）
- LLM 在此层的作用：生成"物种发现记录"和"生态事件"

## 具体接入步骤（结合升级计划）

### 第一步（可以马上做）：验证 Agnes AI + 搭建 LLM client

新建 `src/llm/llm-client.ts`：

```typescript
// 封装 Agnes AI API
// 支持 text completion
// 支持 tool calling（未来用）
// 支持流式（未来用）
// 硬性限制：每 10 秒最多 1 次调用（免费账户 rpm 限制）
// 批量调用先排队，控制速率
```

### 第二步（Phase 0+1 之后）：叙事层

新建 `src/llm/narrative-engine.ts`：
- 输入：WorldState 的 diff（本季变更摘要）
- 输出：一段中文叙事文本
- 写入：chronicle

测试：运行 100 年，比较 LLM 叙事 vs 模板叙事的可读性

### 第三步（Phase 2 之后）：对话层

新建 `src/llm/dialogue-engine.ts`：
- 输入：两个 agent 的 profile + 最近事件
- 输出：一段中文对话
- 写入：agent.memories

### 第四步（Phase 3+4 之后）：事件种子层

新建 `src/llm/event-seeder.ts`：
- 输入：WorldState 的快照
- 输出：3-5 个"微事件种子"，每个带触发条件和后果
- 模拟系统：检查种子条件是否满足，满足则触发

## 成本估算

Agnes AI 免费模型（推测为 agnes-1.5-flash）：

| 层 | 调用频率 | 每次 tokens | 每月估算 |
|----|---------|-------------|---------|
| 叙事层 | 4次/年（每季） | 2K | ~8K tokens |
| 对话层 | 12次/年（每季3对） | 4K | ~48K tokens |
| 事件层 | 4次/年 | 3K | ~12K tokens |
| 总计 | ~20次/年 | ~100K tokens | **几乎免费** |

即使升级到 agnes-2.0-flash（付费模型），按推测定价 ~$0.15/1M tokens → 每月不到 $0.015。

## 重要边界（DOs & DON'Ts）

### DO
- ✅ LLM 生成内容 + 模拟系统决定后果
- ✅ LLM 提供情感和偶然性 + 代码保证一致性
- ✅ 关键角色偶尔引用 LLM + 普通角色全由代码驱动
- ✅ 叙事层可随时关闭（降级到模板）不影响模拟
- ✅ event seed 先写日志再触发（可审查）

### DON'T
- ❌ LLM 直接修改模拟状态（绕过确定性逻辑）
- ❌ 每 agent 每季都调 LLM（成本爆炸 + 延迟）
- ❌ LLM 替代核心模拟机制（经济/生育/交易/冲突概率）
- ❌ LLM 生成的内容作为"ground truth"（防幻觉污染存档）
- ❌ 全自动无审查的 LLM 事件链（防止无限灾难循环）
