# 桃源镇前端 — Stardew Valley 风格可视化计划

> 计划模式 · 仅文档，不执行代码

**目标：** 为桃源镇文明模拟器打造一个星露谷物语风格的 HTML 前端，通过可视化 save file 让文明演化变得可看、可感、可交互。

**核心理念：** 后端仿真引擎产生 `WorldState` JSON，前端读取并渲染。两者通过 save file 解耦——前端是"文明的窗口"，而不是"游戏引擎"。

**风格定位：**
- 像素艺术（16×16/32×32 风格）
- 暖色调、高饱和度、手绘感
- 顶视角 2D 网格地图
- 木质/羊皮纸质感 UI 面板
- 四季颜色体系
- 角色 Q 版头像 + 名字悬浮

---

## 一、数据架构

### 数据流

```
simulation save (last-save.json)
       │
       ▼
index.html  ──loadJSON──▶  Vue/vanilla JS state
       │
       ├─▶ MapRenderer (Canvas/Grid)
       ├─▶ AgentPanel (CSS + DOM)
       ├─▶ ChronicleFeed (CSS + DOM)
       ├─▶ Dashboard (economy + ecology)
       └─▶ SeasonDisplay (overlay + palette)
```

### 模拟状态 → 前端映射

| 模拟数据 | 前端展示 |
|----------|---------|
| `world.map` (TileMap 50×55) | 像素地形网格（Canvas） |
| `agents[]` (27+ 人) | 角色精灵 + 悬浮信息 + 右侧面板 |
| `buildings[]` (11+ 建筑) | 建筑精灵（覆盖在地图上） |
| `year`, `season`, `weather` | 季节指示器 + 整体色调 |
| `economy` | 经济仪表盘（GDP/交易量/物价） |
| `chronicle[]` | 右侧时间线/事件流 |
| `ecology` (污染/资源/生物多样性) | 生态仪表盘 |
| `festivals[]`, `laws[]` | 社会状态面板 |
| `innovations[]` | 科技树展示 |

## 三、组件划分

### Phase 0: 项目骨架（1 个 HTML 文件起步 → 解耦为多文件）

```
town-of-peace/
├── frontend/
│   ├── index.html          ← 主入口（可独立打开）
│   ├── style.css           ← 所有样式
│   ├── app.js              ← 主逻辑 + 路由
│   ├── map-renderer.js     ← 地图 Canvas 渲染
│   ├── agent-panel.js      ← 居民面板
│   ├── chronicle-feed.js   ← 编年史/事件流
│   ├── dashboard.js        ← 仪表盘
│   ├── season-overlay.js   ← 季节覆盖
│   ├── dialog-box.js       ← 对话框系统
│   ├── data-loader.js      ← JSON 加载 + 缓存
│   ├── sprites.js          ← 像素精灵定义
│   └── data/
│       └── last-save.json  ← 模拟存档（软链/拷贝）
```

或保持单 HTML+内联 CSS/JS 的 MVP 模式。

### Phase 1: 地图渲染（视觉基石）

**核心逻辑 `map-renderer.js`：**

1. 读取 `world.map.tiles[][]`（50×55）
2. 每个 TileType 映射到像素颜色/图案：
   - `plains` → 草绿底 + 随机草丛点
   - `forest` → 深绿 + 树木图案
   - `water` → 蓝色 + 波纹动画
   - `mountain` → 灰色 + 岩石纹理
   - `farmland` → 棕色条纹（已耕）或绿色（生长）
   - `road` → 米黄色路径
   - `building` → 预留位置（由 Building 精灵覆盖）
3. Canvas 绘制（`drawImage` 或逐像素 `fillRect`）
4. 支持平移（鼠标拖拽）和缩放（滚轮）

**地图错觉：** 纯 2D 顶视图，不追求 isometric 视角——简化 80% 复杂度，保留像素美感。

### Phase 2: 建筑 + 角色精灵

**建筑 `sprites.js`：**
- 每种建筑类型一种颜色/形状（小房子、工坊、市场等）
- 屋顶用三角形覆盖
- 等级（level）用星级或尺寸表示
- 名称悬浮标签

**角色 `sprites.js`：**
- 每种职业/性别一个简单像素人（16×32）
- 名字悬浮显示
- 鼠标悬停→高亮
- 点击→打开居民详情面板

### Phase 5: 季节 + 天气覆盖

`season-overlay.js`：
- 春季：花瓣飘落粒子效果
- 夏季：阳光闪耀 + 蝉鸣图标

- 导出当前视图为图片
- 全屏模式

- 定时轮询 `last-save.json`（模拟运行时自动刷新）
- 或在后端加 WebSocket 推送
- 自动滚动到最新事件

## 四、技术方案

| 决策 | 选择 | 理由 |
|------|------|------|
| UI 框架 | 无框架 / 原生 JS | 避免依赖，单 HTML 文件可运行 |
| 地图渲染 | Canvas 2D | 性能好，适合像素格渲染 |
| UI 面板 | DOM（HTML + CSS） | 交互性好，文字渲染清晰 |
| 像素精灵 | CSS + Canvas 绘制 | 无需外部图片资源 |
| 数据加载 | `fetch()` 读取 JSON | 简单直接 |
| 可选增强 | Anime.js（粒子动画） | 轻量动画库（可按需引入 CDN） |

### 为什么不用 isometric / 伪 3D？

1. 开发量×3（计算深度排序、遮挡、层级）
2. 桃源镇是文明模拟器，不是动作游戏
3. 顶视图像素风同样可以很美（参考 Stardew Valley 的室内场景）
4. 后续可以从顶视图升级为 45° 等距视图

## 七、风险与权衡

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| save JSON 过大（>10MB） | 低 | 中 | 只加载关键字段，lazy load 编年史 |
| 50×55 格渲染性能 | 低 | 低 | Canvas 离屏渲染 + 视口裁剪 |
| 无框架导致代码组织混乱 | 中 | 中 | 模块化拆分，每个文件单一职责 |
| 像素风格达不到"星露谷"水准 | 中 | 中 | 先做功能，后续用 CSS sprite 替换 Canvas 精灵 |
| 移动端兼容 | 低 | 低 | 先只做桌面端，后续加 responsive |
